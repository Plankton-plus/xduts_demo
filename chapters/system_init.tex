\chapter{初始化}
系统初始化是整个外包检索协议的基石。在数据上传前，DO 必须对明文空间进行代数化映射，构建从高维文本与连续数值到离散有限域元素的映射字典。
该过程涵盖密码学参数生成、关键字词表初始化以及数值前缀空间初始化。
\section{密码学参数生成}
令 $\lambda$ 为系统的安全参数。DO 首先初始化底层密码学原语：

\begin{itemize}
\item \textbf{哈希与伪随机函数：} 选择 BLAKE2b 作为底层哈希算法，实例化伪随机函数 $F: \{0,1\}^\lambda \times \{0,1\}^* \to \{0,1\}^{256}$。随机均匀地抽取一个字符串作为主对称密钥 $mk \leftarrow \{0,1\}^\lambda$。
\item \textbf{累加器参数初始化：} 调用累加器的 $KeyGen(1^\lambda, U)$ 算法，其中 $U$ 为系统支持的最大集合基数。系统随机选取隐藏私钥 $s \leftarrow \mathbb{Z}_p^*$，并基于双线性群 $\mathbb{G}_1, \mathbb{G}_2$ 生成公钥参数元组 $pk$。私钥 $sk$ 由 DO 妥善保管，公钥 $pk$ 公开发布。
\end{itemize}
\section{词表初始化}

为了优化密文检索的存储和计算开销，数据所有者 (DO) 并不直接将定长的哈希令牌作为对象的属性，而是构建一个全局的密文词表 (Keyword Registry)。具体步骤如下：

\begin{itemize}
    \item \textbf{提取与规范化}：DO 扫描所有明文数据对象，提取出关键字集合，并对每个关键字执行规范化操作（如去空格、统一转小写），消除格式差异。
    \item \textbf{令牌生成}：DO 使用主密钥 $mk$，计算每个规范化关键字的 32 字节安全令牌：
    \item \textbf{字典排序与 ID 分配}：DO 收集所有的令牌并进行去重。为了消除令牌生成的时间顺序可能泄露的信息，DO 对所有令牌进行严格的字典序排序。随后，依次为排序后的令牌分配递增的非零短整数 ID（即 \texttt{NonZeroU16}，范围 $1$ 至 $N$）。
\end{itemize}

通过这一设计，我们将任意文本安全且唯一地映射到了有限域 $\mathbb{Z}_p$ 的一个极小子集（即 NonZeroU16 空间）上，完美契合了后续将其作为特征多项式根系的需求。
\section{前缀表初始化}
为了在密文态下支持高效的范围比较（如判断 $x \in [L, R]$），系统引入了前缀编码（Prefix Encoding）范式。DO 将一维数值空间转化为前缀树的节点集合，并对其进行类似于关键字的密文注册。

\begin{itemize}
    \item \textbf{前缀空间构建}：DO 确定数值域的最大比特长度 $L$（如 $L=15$）。系统枚举该长度下所有可能的前缀节点（包括根节点 $\ast$ 到各个叶子节点）。例如，数值 4（二进制 \texttt{100}）的前缀链为 $\{1\ast, 10\ast, 100\}$。
    \item \textbf{前缀加密映射}：与关键字类似，DO 利用主密钥 $mk$ 为枚举出的每一个前缀结构（包含有效位与其长度）生成唯一的令牌：
    \item \textbf{全局前缀表生成}：DO 将所有产生的前缀令牌按字典序排序，并赋予唯一的非零短整数 ID (\texttt{PrefixId})。这一过程构建了全局的前缀注册表 (Prefix Registry)。
\end{itemize}

\section{本章小结}
本章详细介绍了系统初始化阶段的关键步骤，包括密码学参数生成、密文词表初始化以及前缀表初始化。通过规范化、确定性令牌生成和保序扰乱等技术手段，系统成功将高维文本与连续数值映射为离散的有限域元素，为后续的加密索引构建和可验证查询处理奠定了坚实的基础。
